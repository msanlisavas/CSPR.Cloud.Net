# CSPR.Cloud.Net SDK Coverage Audit — 2026-04-22

Source of truth: https://docs.cspr.cloud/ REST API reference + changelog (through v2.9.0, 2026-02-05).

Endpoint line references point to `CSPR.Cloud.Net/Clients/Api/Endpoints.cs` unless otherwise noted. Method line references point to `CSPR.Cloud.Net/Clients/CasperCloudRestClient.cs` (common endpoint region starting at line 273).

## Summary

- Total categories: 32
- Fully covered: 13
- Partially covered (endpoints ok, filters/includers/fields gaps): 15
- Missing entirely: 0
- Categories with missing endpoints: 6 (Account, Validator, Purse, Delegation, Delegator reward, Contract)
- Total missing endpoints: ~9
- Total missing filters: ~7
- Total missing includers: ~10 (most common: `cspr_name` / `rank`)
- Total missing response properties: ~35+ (Deploy is the biggest offender with 8, followed by Bidder, Validator, Supply)

The SDK tracks everything up to the ~v2.4 changelog line cleanly. From v2.4.0 onward, newly added includers (`coingecko_data`, `friendlymarket_data`, `cspr_name`, `rank`, `token_market_data`, `csprtrade_data`) and response fields (annual issuance breakdown, `refund_amount`, `reserved_slots`, `pending_unstaking_amount`, min/max delegation amounts, `bid_amount`) have never landed. Awaiting-deploy, streaming, and the new DEX rate endpoints are the most recent work and are fully wired.

## Cross-cutting observations

### Global type/wire changes not yet honored
1. **String-typed balances (v2.4.3)**: `AccountData.Balance`, `DelegatedBalance`, `UndelegatedBalance`, `StakedBalance` are `ulong?` — API now returns strings. Same risk applies to `BidderData.SelfStake` / `TotalStake` / `DelegatorsStake`, `ValidatorData.SelfStake` / `TotalStake` / `DelegatorsStake`, `SupplyData.Total` / `Circulating`. Model as `string` to avoid silent overflow.
2. **`last_switch_block.timestamp` response wrapper (v2.5.1)**: the paginated response envelope for validator list, supply, auction metrics may have gained this sibling field. Not inspected, not modeled.
3. **CSPR.name (`cspr_name`) includer**: introduced with v2.1.0 across nearly every entity (Account, Validator, Bidder, Deploy caller, Transfer initiator/to/from-purse, FT/NFT ownership/actions, Delegation delegator/validator, Delegator reward). Needs a library-wide sweep.

### Tier-gated and admin endpoints
- Awaiting-deploy is paid-only. The docs note other paid tiers for heavy export endpoints — no SDK change required but users should be aware.

### Streaming (out of audit scope)
Streaming is complete per the latest release (v1.2.0/v1.2.1). The 10 channels in `BaseUrls` (StreamAccountBalances … StreamTransfers) match the docs enumeration. The `caller_hash` filter on the deploys stream (v2.0.12) should be verified against the streaming subscription parameter models under `Parameters/Socket/`.

### Top 10 actionable gaps (recommended implementation order)
1. `DeployData`: add `caller_hash`, `refund_amount`, `consumed_gas`, `version_id`, `pricing_mode_id`, `gas_price_limit`, `is_standard_payment`, `runtime_type_id`, `caller_cspr_name`.
2. `DeploysFilterParameters`: add `caller_hash`.
3. `ValidatorData`: add `bid_amount`, `reserved_slots`, `minimum_delegation_amount`, `maximum_delegation_amount`, `pending_unstaking_amount`, `cspr_name`.
4. `SupplyData`: add `total_annual_issuance` + the two component fields.
5. Era range filters on validator rewards (`from_era_id` / `to_era_id`) + on `AccountDelegatorRewardFilterParameters`.
6. NFT action-type filter (`nft_action_id`) on `NFTContractPackageActionsFilterParameters` and account variant.
7. FT action-type filter (`ft_action_type_id`) on all three FT action filter classes.
8. NFT owner filter (`owner_hash`) on `NFTContractPackageFilterParameters`.
9. Contract-package includers: `token_market_data`, `coingecko_data`, `friendlymarket_data`, `csprtrade_data`, `owner_cspr_name`, plus `website_url` on ContractPackageData.
10. String-typed balance migration on Account/Validator/Bidder/Supply to avoid uint64-overflow deserialization failures.
